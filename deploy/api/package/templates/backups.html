Notas del producto

Ayudas y explicaciones intuitivas para el usuario.

Completo

Crea una copia íntegra de la base de datos seleccionada. Es la opción más recomendable para copias iniciales, pruebas de restauración y puntos de seguridad claros antes de realizar cambios importantes.

Incremental

Guarda únicamente los cambios realizados desde la última copia de referencia. Está pensada para reducir espacio y acelerar procesos, y queda preparada en la interfaz para la evolución completa del sistema.

Diferencial

Genera una copia con los cambios acumulados desde la última copia completa. Es útil cuando se quiere un equilibrio entre rapidez de restauración y menor volumen de almacenamiento.

Funciones relacionadas

Más información para entender la evolución del módulo de copias.

Copias automáticas y programadas

Permite definir copias periódicas para no depender de ejecuciones manuales. La idea es que el sistema pueda mantener una rutina estable de protección según la frecuencia configurada por el usuario.

Restauración

Será la función encargada de recuperar un punto anterior de la base de datos a partir de una copia disponible. Está pensada para resolver errores, pérdidas de información o volver a un estado conocido del sistema.

Historial de copias

Mostrará el registro de copias realizadas con su fecha, tipo, resultado y ubicación. De esta manera el usuario podrá revisar rápidamente qué se ha ejecutado y qué punto de respaldo tiene disponible.
